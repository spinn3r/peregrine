Map Reduce and Bigtable Done Right.

Peregrine is a map reduce framework designed for running iterative jobs across partitions of data. Peregrine is designed to be FAST for executing map reduce jobs by supporting a number of optimizations and features not present in other map reduce frameworks.

Get started today

Fast shuffling

Direct shuffling between hosts which provides for a 2x performance boost over indirect shuffling.

Interative computation.

Many algorithms need to join and merge against data from a previous step. Hadoop implements almost the worst possible strategy for handling joins. With Peregrine, records are stored deterministically so joins are easy and can be done locally. This can provide up to a 50x performance improvement for algorithms like Pagerank and Kmeans.

Lightweight

Peregrine is a lightweight codebase with less than 100k lines of code. This allows it to easily integrate into 3rd party databases and frameworks.

Map/reduce/merge

MapReduceMerge style computations including a new merge() operation allows for clever performance optimizations.

Full async IO

Peregrine is based on Netty, an async IO framework for Java which can scale to a large number of concurrent TCP connections.

Think in map reduce...

Peregrine includes an execution plan optimizer for building jobs at a high level then allowing the runtime to compute the most efficient plan which allows developers to think in higher level constructs rather than being forced to "think" in MapReduce.

Native operating system integration.

Usage of fadvise, mincore, and other operating system interfaces means Peregrine can take advantage of some serious performance optimizations.

Integrated sorting

Peregrine has integrated secondary sorting. The result of a job can be re-sorted by any arbitrary criteria with one line of code.
