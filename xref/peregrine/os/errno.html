161     /**
162      * The routine perror() produces a message on the standard error output,
163      * describing the last error encountered during a call to a system or
164      * library function. First (if s is not NULL and *s is not a null byte
165      * ('\0')) the argument string s is printed, followed by a colon and a
166      * blank. Then the message and a new-line.
167      * 
168      * To be of most use, the argument string should include the name of the
169      * function that incurred the error. The error number is taken from the
170      * external variable errno, which is set when errors occur but not cleared
171      * when non-erroneous calls are made.
172      * 
173      * The global error list sys_errlist[] indexed by errno can be used to
174      * obtain the error message without the newline. The largest message number
175      * provided in the table is sys_nerr -1. Be careful when directly accessing
176      * this list because new error values may not have been added to
177      * sys_errlist[].
178      * 
179      * When a system call fails, it usually returns -1 and sets the variable
180      * errno to a value describing what went wrong. (These values can be found
181      * in <errno.h>.) Many library functions do likewise. The function perror()
182      * serves to translate this error code into human-readable form. Note that
183      * errno is undefined after a successful library call: this call may well
184      * change this variable, even though it succeeds, for example because it
185      * internally used some other library function that failed. Thus, if a
186      * failing call is not immediately followed by a call to perror(), the value
187      * of errno should be saved.
188      */
189     public static int perror( String s ) {
190         return delegate.perror( s );
191     }

193     /**
194      * The strerror() function returns a string describing the error code passed
195      * in the argument errnum, possibly using the LC_MESSAGES part of the
196      * current locale to select the appropriate language. This string must not
197      * be modified by the application, but may be modified by a subsequent call
198      * to perror() or strerror(). No library function will modify this string.
199      * 
200      * The strerror_r() function is similar to strerror(), but is thread
201      * safe. This function is available in two versions: an XSI-compliant
202      * version specified in POSIX.1-2001, and a GNU-specific version (available
203      * since glibc 2.0). If _XOPEN_SOURCE is defined with the value 600, then
204      * the XSI-compliant version is provided, otherwise the GNU-specific version
205      * is provided.
206      * 
207      * The XSI-compliant strerror_r() is preferred for portable applications. It
208      * returns the error string in the user-supplied buffer buf of length
209      * buflen.
210      * 
211      * The GNU-specific strerror_r() returns a pointer to a string containing
212      * the error message. This may be either a pointer to a string that the
213      * function stores in buf, or a pointer to some (immutable) static string
214      * (in which case buf is unused). If the function stores a string in buf,
215      * then at most buflen bytes are stored (the string may be truncated if
216      * buflen is too small) and the string always includes a terminating null
217      * byte.
218      *
219      */
220     public static String strerror( int errnum ) {
221         return delegate.strerror( errnum );
222     }

228     /**
229      * The <errno.h> header file defines the integer variable errno, which is
230      * set by system calls and some library functions in the event of an error
231      * to indicate what went wrong. Its value is significant only when the call
232      * returned an error (usually -1), and a function that does succeed is
233      * allowed to change errno.
234      * 
235      * Sometimes, when -1 is also a valid successful return value one has to
236      * zero errno before the call in order to detect possible errors.
237      * 
238      * errno is defined by the ISO C standard to be a modifiable lvalue of type
239      * int, and must not be explicitly declared; errno may be a macro. errno is
240      * thread-local; setting it in one thread does not affect its value in any
241      * other thread.
242      * 
243      * Valid error numbers are all non-zero; errno is never set to zero by any
244      * library function. All the error names specified by POSIX.1 must have
245      * distinct values, with the exception of EAGAIN and EWOULDBLOCK, which may
246      * be the same.
247      * 
248      * Below is a list of the symbolic error names that are defined on
249      * Linux. Some of these are marked POSIX.1, indicating that the name is
250      * defined by POSIX.1-2001, or C99, indicating that the name is defined by
251      * C99.
252      *
253      */
254     public static int errno() {
255         return Native.getLastError();
256     }
