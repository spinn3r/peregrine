46      /**
47       * 
48       * 
49       * mmap() creates a new mapping in the virtual address space of the calling
50       * process. The starting address for the new mapping is specified in
51       * addr. The length argument specifies the length of the mapping.
52       * 
53       * If addr is NULL, then the kernel chooses the address at which to create
54       * the mapping; this is the most portable method of creating a new
55       * mapping. If addr is not NULL, then the kernel takes it as a hint about
56       * where to place the mapping; on Linux, the mapping will be created at a
57       * nearby page boundary. The address of the new mapping is returned as the
58       * result of the call.
59       * 
60       * The contents of a file mapping (as opposed to an anonymous mapping; see
61       * MAP_ANONYMOUS below), are initialized using length bytes starting at
62       * offset offset in the file (or other object) referred to by the file
63       * descriptor fd. offset must be a multiple of the page size as returned by
64       * sysconf(_SC_PAGE_SIZE).
65       * 
66       * The prot argument describes the desired memory protection of the mapping
67       * (and must not conflict with the open mode of the file). It is either
68       * PROT_NONE or the bitwise OR of one or more of the following flags:
69       * 
70       * PROT_EXEC
71       * 
72       * Pages may be executed.
73       * 
74       * PROT_READ Pages may be read.
75       * 
76       * PROT_WRITE
77       * 
78       * Pages may be written.
79       * 
80       * PROT_NONE
81       * 
82       * Pages may not be accessed.
83       * 
84       * The flags argument determines whether updates to the mapping are visible
85       * to other processes mapping the same region, and whether updates are
86       * carried through to the underlying file. This behavior is determined by
87       * including exactly one of the following values in flags:
88       * 
89       * MAP_SHARED
90       * 
91       * Share this mapping. Updates to the mapping are visible to other processes
92       * that map this file, and are carried through to the underlying file. The
93       * file may not actually be updated until msync(2) or munmap() is called.
94       * 
95       * MAP_PRIVATE
96       * 
97       * Create a private copy-on-write mapping. Updates to the mapping are not
98       * visible to other processes mapping the same file, and are not carried
99       * through to the underlying file. It is unspecified whether changes made to
100      * the file after the mmap() call are visible in the mapped region.
101      * 
102      * Both of these flags are described in POSIX.1-2001.
103      * 
104      * In addition, zero or more of the following values can be ORed in flags:
105      * 
106      * MAP_32BIT (since Linux 2.4.20, 2.6)
107      * 
108      * Put the mapping into the first 2 Gigabytes of the process address
109      * space. This flag is only supported on x86-64, for 64-bit programs. It was
110      * added to allow thread stacks to be allocated somewhere in the first 2GB
111      * of memory, so as to improve context-switch performance on some early
112      * 64-bit processors. Modern x86-64 processors no longer have this
113      * performance problem, so use of this flag is not required on those
114      * systems. The MAP_32BIT flag is ignored when MAP_FIXED is set.
115      * 
116      * MAP_ANON
117      * 
118      * Synonym for MAP_ANONYMOUS. Deprecated.
119      * 
120      * MAP_ANONYMOUS
121      * 
122      * The mapping is not backed by any file; its contents are initialized to
123      * zero. The fd and offset arguments are ignored; however, some
124      * implementations require fd to be -1 if MAP_ANONYMOUS (or MAP_ANON) is
125      * specified, and portable applications should ensure this. The use of
126      * MAP_ANONYMOUS in conjunction with MAP_SHARED is only supported on Linux
127      * since kernel 2.4.
128      * 
129      * MAP_DENYWRITE
130      * 
131      * This flag is ignored. (Long ago, it signaled that attempts to write to
132      * the underlying file should fail with ETXTBUSY. But this was a source of
133      * denial-of-service attacks.)
134      * 
135      * MAP_EXECUTABLE
136      * 
137      * This flag is ignored.
138      * 
139      * MAP_FILE
140      * 
141      * Compatibility flag. Ignored.
142      * 
143      * MAP_FIXED
144      * 
145      * Don't interpret addr as a hint: place the mapping at exactly that
146      * address. addr must be a multiple of the page size. If the memory region
147      * specified by addr and len overlaps pages of any existing mapping(s), then
148      * the overlapped part of the existing mapping(s) will be discarded. If the
149      * specified address cannot be used, mmap() will fail. Because requiring a
150      * fixed address for a mapping is less portable, the use of this option is
151      * discouraged.
152      * 
153      * MAP_GROWSDOWN
154      * 
155      * Used for stacks. Indicates to the kernel virtual memory system that the
156      * mapping should extend downward in memory.
157      * 
158      * MAP_HUGETLB (since Linux 2.6.32)
159      * 
160      * Allocate the mapping using "huge pages." See the kernel source file
161      * Documentation/vm/hugetlbpage.txt for further information.
162      * 
163      * MAP_LOCKED (since Linux 2.5.37)
164      * 
165      * Lock the pages of the mapped region into memory in the manner of
166      * mlock(2). This flag is ignored in older kernels.
167      * 
168      * MAP_NONBLOCK (since Linux 2.5.46)
169      * 
170      * Only meaningful in conjunction with MAP_POPULATE. Don't perform
171      * read-ahead: only create page tables entries for pages that are already
172      * present in RAM. Since Linux 2.6.23, this flag causes MAP_POPULATE to do
173      * nothing. One day the combination of MAP_POPULATE and MAP_NONBLOCK may be
174      * reimplemented.
175      * 
176      * MAP_NORESERVE
177      * 
178      * Do not reserve swap space for this mapping. When swap space is reserved,
179      * one has the guarantee that it is possible to modify the mapping. When
180      * swap space is not reserved one might get SIGSEGV upon a write if no
181      * physical memory is available. See also the discussion of the file
182      * /proc/sys/vm/overcommit_memory in proc(5). In kernels before 2.6, this
183      * flag only had effect for private writable mappings.
184      * 
185      * MAP_POPULATE (since Linux 2.5.46) Populate (prefault) page tables for a
186      * mapping. For a file mapping, this causes read-ahead on the file. Later
187      * accesses to the mapping will not be blocked by page faults. MAP_POPULATE
188      * is only supported for private mappings since Linux 2.6.23.
189      * 
190      * MAP_STACK (since Linux 2.6.27)
191      * 
192      * Allocate the mapping at an address suitable for a process or thread
193      * stack. This flag is currently a no-op, but is used in the glibc threading
194      * implementation so that if some architectures require special treatment
195      * for stack allocations, support can later be transparently implemented for
196      * glibc.
197      * 
198      * MAP_UNINITIALIZED (since Linux 2.6.33)
199      * 
200      * Don't clear anonymous pages. This flag is intended to improve performance
201      * on embedded devices. This flag is only honored if the kernel was
202      * configured with the CONFIG_
203      * 
204      * MMAP_ALLOW_UNINITIALIZED option. Because of the security implications,
205      * that option is normally enabled only on embedded devices (i.e., devices
206      * where one has complete control of the contents of user memory).
207      * 
208      * Of the above flags, only MAP_FIXED is specified in POSIX.1-2001. However,
209      * most systems also support MAP_ANONYMOUS (or its synonym MAP_ANON).
210      * 
211      * Some systems document the additional flags MAP_AUTOGROW, MAP_AUTORESRV,
212      * MAP_COPY, and MAP_LOCAL.
213      * 
214      * Memory mapped by mmap() is preserved across fork(2), with the same
215      * attributes.
216      * 
217      * A file is mapped in multiples of the page size. For a file that is not a
218      * multiple of the page size, the remaining memory is zeroed when mapped,
219      * and writes to that region are not written out to the file. The effect of
220      * changing the size of the underlying file of a mapping on the pages that
221      * correspond to added or removed regions of the file is unspecified.
222      *
223      * 
224      * Return Value
225      * 
226      * On success, mmap() returns a pointer to the mapped area. On error, the
227      * value MAP_FAILED (that is, (void *) -1) is returned, and errno is set
228      * appropriately. On success, munmap() returns 0, on failure -1, and errno
229      * is set (probably to EINVAL).
230      * 
231      */
232     public static Pointer mmap( long len, int prot, int flags, int fildes, long off )
233         throws IOException {
234 
235         // we don't really have a need to change the recommended pointer.
236         Pointer addr = new Pointer( 0 );
237         
238         Pointer result = Delegate.mmap( addr, len, prot, flags, fildes, off );
239         
240         if ( Pointer.nativeValue( result ) == -1 ) {
241             throw new IOException( errno.strerror() );
242         }
243 
244         return result;
245         
246     }

248     /**
249      * The munmap() system call deletes the mappings for the specified address
250      * range, and causes further references to addresses within the range to
251      * generate invalid memory references. The region is also automatically
252      * unmapped when the process is terminated. On the other hand, closing the
253      * file descriptor does not unmap the region.
254      * 
255      * The address addr must be a multiple of the page size. All pages
256      * containing a part of the indicated range are unmapped, and subsequent
257      * references to these pages will generate SIGSEGV. It is not an error if
258      * the indicated range does not contain any mapped pages.
259      */
260     public static int munmap( Pointer addr, long len )
261         throws IOException {
262 
263         int result = Delegate.munmap( addr, len );
264 
265         if ( result != 0 ) {
266             throw new IOException( errno.strerror() );
267         }
268 
269         return result;
270 
271     }

273     /**
274      * mlock() and mlockall() respectively lock part or all of the calling
275      * process's virtual address space into RAM, preventing that memory from
276      * being paged to the swap area. munlock() and munlockall() perform the
277      * converse operation, respectively unlocking part or all of the calling
278      * process's virtual address space, so that pages in the specified virtual
279      * address range may once more to be swapped out if required by the kernel
280      * memory manager. Memory locking and unlocking are performed in units of
281      * whole pages.
282      * 
283      * mlock() and munlock()
284      * 
285      * mlock() locks pages in the address range starting at addr and continuing
286      * for len bytes. All pages that contain a part of the specified address
287      * range are guaranteed to be resident in RAM when the call returns
288      * successfully; the pages are guaranteed to stay in RAM until later
289      * unlocked.
290      * 
291      * munlock() unlocks pages in the address range starting at addr and
292      * continuing for len bytes. After this call, all pages that contain a part
293      * of the specified memory range can be moved to external swap space again
294      * by the kernel.
295      * 
296      */
297     public static void mlock( Pointer addr, long len )
298         throws IOException {
299 
300         if ( Delegate.mlock( addr, len ) != 0 ) {
301             PlatformException cause = new PlatformException();
302             throw new IOException( String.format( "Unable to mlock %s with length %s: %s" , addr, len, cause.getMessage() ), cause );
303         }
304 
305     }
306 
307     /**
308      * Unlock the given region, throw an IOException if we fail.
309      */
310     public static void munlock( Pointer addr, long len )
311         throws IOException {
312 
313         Platform.assertResult( Delegate.munlock( addr, len ) );
314 
315     }

317     /**
318      * Return information about the VFS page cache.  Specifically if given pages
319      * are cached.  This method is fast, even or a large file.  About 1ms for a
320      * 500MB file on Linux.
321      *
322      * @param addr The pointer in memory of the mmap region as returned from mmap.
323      * @param len The length of the file for which we want page cache information.
324      * @return a ByteBuffer where the offset is true if the given page is cached.
325      * @throws IOException
326      */
327     public static ByteBuffer mincore( Pointer addr, long len ) throws IOException {
328 
329         int page_size = unistd.getpagesize();
330 
331         int buff_size = (int)((len + page_size - 1) / page_size);
332 
333         ByteBuffer vec = ByteBuffer.allocate( buff_size );
334 
335         int result = Delegate.mincore( addr, len, vec );
336         
337         Platform.assertResult( result );
338 
339         return vec;
340 
341     }

358     public static void main( String[] args ) throws Exception {
359 
360         String path = args[0];
361 
362         File file = new File( path );
363         FileInputStream in = new FileInputStream( file );
364         int fd = Platform.getFd( in.getFD() );
365 
366         // mmap a large file... 
367         Pointer addr = mmap( file.length(), PROT_READ, mman.MAP_SHARED | mman.MAP_LOCKED, fd, 0 );
368 
369         // try to mlock it directly
370         mlock( addr, file.length() );
371         munlock( addr, file.length() );
372 
373         ByteBuffer vec = null;
374 
375         long before = System.currentTimeMillis();
376         long max = 1000000;
377         for( long i = 0; i < max; ++i ) {
378             vec = mincore( addr, 1 );
379         }
380         long after = System.currentTimeMillis();
381 
382         long duration = after - before;
383 
384         System.out.printf( "mincore duration: %,d ms for %,d calls.\n", duration, max );
385         
386         System.out.printf( "First(2) block cached: %s\n", vec.get(0) );
387 
388         munmap( addr, file.length() );
389 
390     }
