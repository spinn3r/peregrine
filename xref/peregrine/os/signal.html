56      /**
57       * The kill() system call can be used to send any signal to any process
58       * group or process.
59       * 
60       * If pid is positive, then signal sig is sent to pid.
61       * 
62       * If pid equals 0, then sig is sent to every process in the process group
63       * of the current process.
64       * 
65       * If pid equals -1, then sig is sent to every process for which the calling
66       * process has permission to send signals, except for process 1 (startup), but
67       * see below.
68       * 
69       * If pid is less than -1, then sig is sent to every process in the process
70       * group -pid.
71       * 
72       * If sig is 0, then no signal is sent, but error checking is still
73       * performed.
74       * 
75       * For a process to have permission to send a signal it must either be
76       * privileged (under Linux: have the CAP_KILL capability), or the real or
77       * effective user ID of the sending process must equal the real or saved
78       * set-user-ID of the target process. In the case of SIGCONT it suffices
79       * when the sending and receiving processes belong to the same session.
80       * Return Value On success (at least one signal was sent), zero is
81       * returned. On error, -1 is returned, and errno is set appropriately.
82       * Errors
83       * 
84       * EINVAL
85       * An invalid signal was specified. 
86       * 
87       * EPERM
88       * The process does not have permission to send the signal to any of the target processes. 
89       * 
90       * ESRCH
91       * 
92       * The pid or process group does not exist. Note that an existing process
93       * might be a zombie, a process which already committed termination, but has
94       * not yet been wait()ed for.
95       * 
96       * Notes
97       * 
98       * The only signals that can be sent task number one, the startup process, are
99       * those for which startup has explicitly installed signal handlers. This is
100      * done to assure the system is not brought down accidentally.
101      * 
102      * POSIX.1-2001 requires that kill(-1,sig) send sig to all processes that
103      * the current process may send signals to, except possibly for some
104      * implementation-defined system processes. Linux allows a process to signal
105      * itself, but on Linux the call kill(-1,sig) does not signal the current
106      * process.
107      * 
108      * POSIX.1-2001 requires that if a process sends a signal to itself, and the
109      * sending thread does not have the signal blocked, and no other thread has
110      * it unblocked or is waiting for it in sigwait(), at least one unblocked
111      * signal must be delivered to the sending thread before the kill().
112      * 
113      * Bugs
114      * 
115      * In 2.6 kernels up to and including 2.6.7, there was a bug that meant that
116      * when sending signals to a process group, kill() failed with the error
117      * EPERM if the caller did have permission to send the signal to any (rather
118      * than all) of the members of the process group. Notwithstanding this error
119      * return, the signal was still delivered to all of the processes for which
120      * the caller had permission to signal.
121      * 
122      * Linux History
123      * 
124      * Across different kernel versions, Linux has enforced different rules for
125      * the permissions required for an unprivileged process to send a signal to
126      * another process. In kernels 1.0 to 1.2.2, a signal could be sent if the
127      * effective user ID of the sender matched that of the receiver, or the real
128      * user ID of the sender matched that of the receiver. From kernel 1.2.3
129      * until 1.3.77, a signal could be sent if the effective user ID of the
130      * sender matched either the real or effective user ID of the receiver. The
131      * current rules, which conform to POSIX.1-2001, were adopted in kernel
132      * 1.3.78.
133      */
134     public static int kill( int pid, int sig ) throws PlatformException {
135 
136         int result = Delegate.kill( pid, sig );
137 
138         if ( result == -1 )
139             throw new PlatformException();
140 
141         return result;
142 
143     }
